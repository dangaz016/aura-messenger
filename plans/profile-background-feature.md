# План реализации кастомных фонов профиля

## Обзор
Добавление кастомных фонов профиля как премиум-фичи, позволяющей пользователям персонализировать заголовок своего профиля. Это может включать заранее подготовленные градиентные фоны или пользовательские изображения.

## Изменения в базе данных

1. Добавить колонку в таблицу users:
```sql
ALTER TABLE users ADD COLUMN prime_profile_background TEXT DEFAULT '';
```

2. Эта колонка будет хранить:
   - Пустая строка: градиент по умолчанию на основе цвета аватара
   - ID пресета (например, "gradient1", "gradient2"): для заранее определённых фонов
   - URL: для пользовательских загруженных изображений

## Изменения в бэкенд API

1. Обновить endpoint PATCH /api/prime/settings в prime.ts:
   - Добавить profile_background как принимаемый параметр
   - Добавить валидацию для допустимых значений (пустая строка, ID пресетов, валидные URL)

2. Необходимая валидация:
   - Пустая строка: допустимо (сброс к значению по умолчанию)
   - ID пресетов: должны быть из заранее определённого набора (gradient1, gradient2 и т.д.)
   - URL: должны быть из разрешённых доменов или файлового сервиса

## Реализация фронтенда

1. Создать компонент BackgroundSelector с:
   - Миниатюрами заранее подготовленных градиентов
   - Опцией загрузки изображения (для Prime пользователей)
   - Функционалом предварительного просмотра

2. Добавить этот компонент в AuraPrimePanel.tsx в секцию настройки профиля

3. Обновить UserProfile.tsx для:
   - Проверки user.prime_profile_background
   - Применения соответствующих стилей фона к заголовку профиля

## Соображения по UI дизайну

1. Опции заранее подготовленных фонов должны быть визуально согласованы с дизайн-системой приложения
2. Пользовательские изображения должны быть правильно масштабированы и оптимизированы для производительности
3. Фон не должен мешать читаемости информации профиля
4. Добавить индикатор, показывающий, что это функция только для Prime

## Обработка ошибок и особые случаи

1. Некорректные настройки фона должны корректно откатываться к значению по умолчанию
2. Битые URL изображений не должны ломать UI
3. Фоны должны быть адаптивными для разных размеров экранов
4. Обеспечить приемлемую производительность для анимированных/кастомных фонов
